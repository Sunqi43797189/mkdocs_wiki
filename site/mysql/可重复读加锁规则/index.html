    作者总结

    • 个人学习

    两个原则

    • 加锁的基本单位是next-key-lock，前开后闭的区间
    • 查询过程中访问到的对象才会加锁

    两个优化

    • 索引上的等值查询，给唯一索引加锁的后next-key-lock会退化成行锁
    • 索引上的等值查询，向右遍历时且最后一个值不满足等值条件的时候，next-key-lock退化为间隙锁

    一个bug

    • 唯一索引上的范围查询会访问到不满足条件的第一个值为止

    是否锁主键索引

    • 覆盖索引的情况lock in share mode 不会锁主键索引
    • select * for update 会锁主键
